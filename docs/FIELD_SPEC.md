# 数据字段说明

本文档基于实际GDB图层分析结果生成，详细说明1:100万基础地理信息数据中所有表的字段含义，帮助LLM正确理解和使用字段。

## 说明

- **大部分字段说明**基于F49和G49图幅的实际数据分析生成，包含空值率统计和示例值
- **部分图层**（RESL、VEGL、PIPP）在分析图幅中未找到数据，字段说明基于数据规格推断
- 如需更完整的字段说明，建议分析更多图幅的GDB文件

## 通用字段

所有表都包含以下通用字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | INTEGER | 主键，自动递增的唯一标识符 |
| `geom` | GEOMETRY | PostGIS几何对象，存储空间数据（点、线、面） |
| `tile_code` | VARCHAR(10) | 图幅代码，1:100万图幅编号（如F49、F50、G49、G50等） |

### 境界与政区

#### boua (行政境界(面))

**几何类型**: MultiPolygon

**重要特点**：
1. **数据层级**：boua表只包含区、县、县级市等最小行政单元，**不包含地级市**数据
2. **数据分割**：同一个行政区域可能被分割成多个记录（多个行），原因包括：
   - 跨越多个图幅（如F49和F50）
   - 被其他地理要素（如河流、道路）分割
   - 数据本身的组织方式
3. **查询方法**：查询某个行政区域时，需要将所有相同`name`或`pac`的记录合并视为一个整体，使用`ST_Union`合并几何，使用`SUM`计算总面积

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `name` | VARCHAR(60) | 名称字段，存储要素的中文名称（可能为空）（空值率3.6%），用于标识行政区域名称 |
| `pac` | INTEGER | 属性分类代码（Property Attribute Code），用于标识要素的分类，相同行政区域的记录通常有相同的pac值 |
| `shape_area` | DOUBLE PRECISION | **注意**：几何形状的面积，单位为**度²**（平方度）。**不能直接转换为平方公里**，因为度²到平方公里的转换不是线性的（在不同纬度下，1度²对应的实际面积不同）。**应使用`ST_Area(geom::geography)`计算实际面积**（返回平方米，除以1000000得到平方公里）。此字段仅用于参考，实际计算面积时请使用PostGIS的ST_Area函数。 |
| `shape_length` | DOUBLE PRECISION | **注意**：几何形状的边界长度，单位为**度**。**不能直接转换为公里**，因为度到公里的转换不是线性的（在不同纬度下，1度对应的实际长度不同）。**应使用`ST_Length(geom::geography)`计算实际长度**（返回米，除以1000得到公里）。此字段仅用于参考，实际计算长度时请使用PostGIS的ST_Length函数。 |

**查询示例**：
```sql
-- 查询香洲区的完整面积（合并所有分割的部分）
SELECT 
    name,
    pac,
    COUNT(*) as part_count,  -- 该行政区域被分割成几部分
    ST_Area(ST_Union(geom)::geography) / 1000000 as total_area_km2,
    ST_AsText(ST_Centroid(ST_Union(geom))) as centroid
FROM boua
WHERE name = '香洲区'
GROUP BY name, pac;

-- 或者使用pac代码查询（如果name为空）
SELECT 
    pac,
    MAX(name) as name,  -- 如果有name值
    COUNT(*) as part_count,
    ST_Area(ST_Union(geom)::geography) / 1000000 as total_area_km2
FROM boua
WHERE pac = 440402
GROUP BY pac;
```
